## 一、三句一锤定音

以下三组格言级句子，每句都能在标题幻灯上单独成立。

1. **"一个进程，一个意识。"**
   每个灵台 agent 是一个独立的长运行进程，不是一次性的 API 调用。

2. **"化身即网络。"**
   Agent 可以化出子 agent，形成树状网络，通过飞鸽（邮件）通信。

3. **"凝蜕即重生。"**
   Agent 主动蜕去对话历史，保留记忆与身份，以全新上下文继续工作。

## 四、避免之雷区

> **以下内容绝对不要出现在幻灯中。**

1. **不要 over-claim AGI**：灵台不是通用人工智能，不做这种暗示。定位是"AI Agent 运行时"。
2. **不要轻言比 OpenAI / Anthropic 强**：灵台是不同物种，不是谁的替代品。如需对比，用"物种之异"而非"优劣之争"。
3. **不要暗示 agent 有真正的自我意识或情感**：说"机制"（mechanism），不说"感觉"（feeling）。soul 是"内省机制"，不是"灵魂"。
4. **不要拿未经确认的数据做噱头**：所有"待确认"项不要出现在最终幻灯中。
5. **不要在技术细节上停留太久**：幻灯是钩子，不是文档。每个概念一句话带过，深入留给 live demo。
6. **不要忽略文言幻灯的仪式感**：Slide 4（言之择）是情感锚点，不宜删减或过度理性化。信约原文的展示本身就是内容。
7. **不要将灵台与 pi-mono 做零和对比**：vs-pi-mono.md 已明确定调——"非优劣之争，乃物种之异"。Pi-mono 做了一件事并做得极好；灵台做的是另一件事。

## 五、Token 过剩时代——论据材料

> 供 huangzs 在钩子幻灯或物种命题幻灯中引用。

### 核心论点

所有现存 agent 框架（LangChain、pi-mono、Assistants）皆基于 **"token 稀缺"假设**——节省、压缩、淘汰。但今日极客之实情是 **token 已经过剩**，却无框架告诉他们如何**有结构地烧**。

LingTai 的回答：**token 不是成本，是原材料。** 化身五身齐出，每身自烧自之 context，但合力压一深题——这就是"有结构地烧"。

> **稀缺时代的设计**：一个 agent，省着用，用完即弃。
> **过剩时代的设计**：一个网络，放开烧，烧出结构。
